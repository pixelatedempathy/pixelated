## Complete Method Catalog

### 0. Foundational Method (Use FIRST)

#### [Problem Definition Thinking](problem-definition.md)
- **Purpose**: Clearly define and clarify the true problem before solving
- **Best for**: ANY major problem-solving effort (use before other methods)
- **Time**: Quick (30-60 minutes for complex problems)
- **Success rate**: 100% for preventing wasted effort on wrong problems
- **Key Principle**: "All problem-solving begins with correctly defining the problem"

---

### 1. Root Cause Analysis Methods

#### [5 Why Technique](5-why.md)
- **Purpose**: Find root cause through repeated "why" questions
- **Best for**: Single causal chain problems
- **Time**: Quick (5-15 minutes)
- **Success rate**: 100% for root_cause_analysis

#### [Fishbone Diagram (Ishikawa)](fishbone.md)
- **Purpose**: Structured multi-dimensional cause analysis
- **Best for**: Complex problems with multiple factors
- **Time**: Medium (20-40 minutes)
- **Success rate**: 100% for root_cause_analysis

---

### 2. Innovation & Breakthrough Methods

#### [First Principles Thinking](first-principles.md)
- **Purpose**: Fundamental reconstruction from basic truths
- **Best for**: Paradigm-shifting innovation
- **Time**: Long (1-2 hours)
- **Success rate**: 100% for technical_problem

#### [Design Thinking](design-thinking.md)
- **Purpose**: User-centric innovation process
- **Best for**: Human-centered problem solving
- **Time**: Long (days to weeks for full cycle)
- **Success rate**: 100% for creative_innovation

#### [TRIZ](triz.md)
- **Purpose**: Technical contradiction resolution
- **Best for**: Engineering innovations
- **Time**: Medium (30-60 minutes)
- **Success rate**: 100% for creative_innovation

#### [SCAMPER](scamper.md)
- **Purpose**: Creative modification prompts
- **Best for**: Quick ideation and product improvements
- **Time**: Quick (15-30 minutes)
- **Success rate**: 100% for creative_innovation

---

### 3. Strategic Planning Methods

#### [SWOT Analysis](swot.md)
- **Purpose**: Strengths, Weaknesses, Opportunities, Threats
- **Best for**: Strategic situation assessment
- **Time**: Medium (30-45 minutes)
- **Success rate**: 75% (⚠️ MUST add 2x2 priority matrix)

#### [GAP Analysis](gap-analysis.md)
- **Purpose**: Current → Target state planning
- **Best for**: Goal-oriented execution planning
- **Time**: Medium (20-40 minutes)
- **Success rate**: 100% for strategic_planning, process_improvement

---

### 4. Process Improvement Methods

#### [Pareto Analysis (80/20 Rule)](pareto.md)
- **Purpose**: Identify critical 20% causing 80% impact
- **Best for**: Data-driven prioritization
- **Time**: Medium (30-45 minutes with data)
- **Success rate**: 100% for process_improvement

#### [PDCA Cycle](pdca.md)
- **Purpose**: Iterative improvement (Plan-Do-Check-Act)
- **Best for**: Continuous improvement initiatives
- **Time**: Ongoing cycles
- **Success rate**: 100% for process_improvement

#### [DMAIC (Six Sigma)](dmaic.md)
- **Purpose**: Rigorous quality improvement
- **Best for**: Manufacturing and quality control
- **Time**: Long (weeks to months)
- **Success rate**: High for quality control
- **⚠️ Warning**: NOT for creative_innovation (0% success)

---

### 5. Decision-Making Methods

#### [OODA Loop](ooda-loop.md)
- **Purpose**: Rapid decision cycle (Observe-Orient-Decide-Act)
- **Best for**: Fast-paced, dynamic environments
- **Time**: Quick iterations (minutes to hours per cycle)
- **Success rate**: 100% for decision_making

#### [Kepner-Tregoe](kepner-tregoe.md)
- **Purpose**: Systematic decision and problem analysis
- **Best for**: High-stakes critical decisions
- **Time**: Long (1-2 hours)
- **Success rate**: Effective for systematic decisions
- **⚠️ Warning**: NOT for creative_innovation (0% success)

---

### 6. Synthesis & Integration Methods

#### [Dialectic Synthesis](dialectic.md)
- **Purpose**: Thesis-Antithesis-Synthesis integration
- **Best for**: Resolving opposing viewpoints
- **Time**: Medium (30-60 minutes)
- **⚠️ Warning**: NOT for simple problems (over-engineering)

## Method Combinations

Many complex problems benefit from combining methods:

### Recommended Combinations

**Complex System Failure**:
1. [5 Why](5-why.md) → Find root cause
2. [Fishbone](fishbone.md) → Identify contributing factors
3. [Kepner-Tregoe](kepner-tregoe.md) → Systematic decision on fix
4. [PDCA](pdca.md) → Implement and validate

**New Product Strategy**:
1. [Design Thinking](design-thinking.md) → User research & ideation
2. [SWOT](swot.md) → Market analysis
3. [First Principles](first-principles.md) → Challenge assumptions
4. [GAP Analysis](gap-analysis.md) → Execution roadmap
5. [OODA Loop](ooda-loop.md) → Rapid MVP iteration

**Organizational Change**:
1. [SWOT](swot.md) → Current situation
2. [GAP Analysis](gap-analysis.md) → Transformation goals
3. [Kepner-Tregoe](kepner-tregoe.md) → Decision on approach
4. [PDCA](pdca.md) → Pilot implementation
5. [DMAIC](dmaic.md) → Scaling & standardization

## Recommended Workflow

For ANY problem-solving effort, follow this sequence:

1. **[Problem Definition](problem-definition.md)** ← START HERE (30-60 min)
   - Clarify what the real problem is
   - Uncover hidden assumptions
   - Define success criteria

2. **Select Appropriate Method** (based on problem type)
   - Use [Problem Type Selector](#by-problem-type) above
   - Apply selected method from this catalog

3. **Execute & Validate**
   - Follow method steps
   - Measure outcomes against success criteria
   - Reflect and iterate

**This two-step approach prevents the most common failure: solving the wrong problem efficiently.**
