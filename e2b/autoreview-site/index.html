FAQ

Common questions.

Is my code safe?

Yes. All fixes run in an isolated E2B sandbox ‚Äî a disposable virtual machine that's destroyed after each run. Your code is cloned into the sandbox, fixed, and pushed. It never touches shared infrastructure or other users' workloads.

What if the AI generates a bad fix?

Every fix runs through your linter, type checker, and test suite inside the sandbox before committing. If any check fails, the fix is discarded. You also review the resulting PR diff like any other commit.

What review comments can it fix?

AutoReview handles most common review feedback: unused imports, type annotations, null checks, code style, refactoring suggestions, error handling, and more. It works best on concrete, actionable comments. Subjective architecture debates are left to humans.

Does it work with GitLab / Bitbucket?

Currently GitHub only. GitLab support is on the roadmap for Q2 2026. Interested? Join the waitlist and we'll notify you.

Can I self-host it?

Enterprise plans include a self-hosted option with your own E2B instances and LLM keys. The core engine is open-source. Contact sales for details.

What languages are supported?

Any language the LLM can reason about ‚Äî TypeScript, JavaScript, Python, Go, Rust, Java, C#, and more. The sandbox runs your actual toolchain (linters, compilers, test runners) for verification.
